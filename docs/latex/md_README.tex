本项目旨在通过设计并实现一个初步的图形渲染系统，综合运用以下9种典型设计模式：


\begin{DoxyItemize}
\item • 创建型模式：
\item ◦ Factory（工厂）
\item ◦ Abstract Factory（抽象工厂）
\item ◦ Singleton（单例）
\item • 结构型模式：
\item ◦ Adapter（适配器）
\item ◦ Bridge（桥接）
\item ◦ Proxy（代理）
\item • 行为型模式：
\item ◦ Command（命令）
\item ◦ Visitor（访问者）
\item ◦ Observer（观察者）
\item 
\end{DoxyItemize}\hypertarget{md_README_autotoc_md2}{}\doxysection{🚀 二、项目目标}\label{md_README_autotoc_md2}
设计并实现一个简化的图形渲染系统，具备以下功能： -\/1. 渲染基本图形元素（如圆形、矩形、线段）。 -\/2. 支持不同的渲染引擎（如 SVG、\+Canvas、终端字符图）。 -\/3. 支持用户操作命令（如添加图形、移动、撤销、重做）。 -\/4. 支持图形访问器（例如导出图形结构为 JSON、\+XML）。 -\/5. 具备插件代理机制（如远程渲染服务）。\hypertarget{md_README_autotoc_md3}{}\doxysection{🚀 三、任务要求}\label{md_README_autotoc_md3}

\begin{DoxyItemize}
\item 1. 功能要求
\item • 图形对象定义（使用抽象工厂+工厂模式创建）
\item • 渲染器接口与实现（使用桥接模式支持多种渲染方式）
\item • 图形命令操作（使用命令模式实现操作和撤销）
\item • 渲染服务代理（使用代理模式本地与远程渲染）
\item • 监听模型变化（使用观察者模式）
\item • 数据导出（访问者模式：导出为\+JSON、\+XML等格式）
\item • 渲染器适配器（适配不同图形库）
\item • 系统全局配置（使用单例模式提供共享配置）
\item 2. 技术要求
\item • 使用 Java（建议 Java 11 或以上）
\item • 使用 Power\+Designer 建模（类图、时序图）
\item • 使用 JUnit 实现测试（单元测试、集成测试）
\item • 使用 Java\+Doc 或 Doxygen 自动生成文档
\item • 提供完整可执行包和源代码
\item
